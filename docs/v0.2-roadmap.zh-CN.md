# Super CDN v0.2 Roadmap

[English](v0.2-roadmap.md) | 简体中文

v0.2 的主题是把 Super CDN 从简单静态对象控制面推进到可发布静态网站、可接资源库、可做初步真实验证的内部版本。

## 产品主题

目标：

- 支持静态网站 lifecycle；
- 引入 site、deployment、domain 的基础模型；
- 保留 R2/AList/IPFS 等 provider 集成方向；
- 形成可维护的维护分支和后续 roadmap。

## 版本边界

v0.2 不是智能路由、自动 failover 或完整 Cloudflare Static 时代的最终形态。它的价值是让站点和 provider 方向可跑通，并把后续风险暴露出来。

## Milestone 清单

| 里程碑 | 内容 |
| --- | --- |
| Site 模型 | `create-site`、`list-sites`、domain、deployment |
| 部署流程 | 从本地目录或 zip 生成不可变 deployment |
| 生产指针 | active deployment 指向当前 production |
| Preview/production 访问 | 支持 preview 和 production 路径 |
| Provider 方向 | R2、AList/OpenList、IPFS/Pinata 的配置承载 |
| 诊断 | 初步 `doctor`、site probe 和错误输出 |
| 文档 | 维护边界、roadmap 和 release note |

## v0.2 非目标

- 不承诺完整智能路由；
- 不做 provider-aware rollback；
- 不把 R2-backed Web hosting 当作长期主流路径；
- 不做大规模 UI；
- 不做自动跨线 failover。

## 早期风险

- Provider 写入和控制面 metadata 可能不一致；
- 真实域名、DNS、证书和 Worker route 需要明确验证；
- AList/OpenList、R2、IPFS 的可见性和 HEAD/GET 行为不同；
- 文档容易把 canary 成功误写成全面成熟。

## v0.2 Definition Of Done

- 基础站点部署和读取跑通；
- 关键 CLI 命令有文档；
- 真实 canary 至少证明一条 Web 路径；
- v0.2 线边界清楚，后续新功能进入 v0.3+。
