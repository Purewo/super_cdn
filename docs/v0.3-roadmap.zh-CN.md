# Super CDN v0.3 Roadmap

[English](v0.3-roadmap.md) | 简体中文

v0.3 的主题是资源能力、复制、副本修复、智能路由和显式 failover 的成型，为后续成熟度周期打基础。

## 产品主题

让 Super CDN 能清楚表达：

- 某条资源线能做什么；
- 对象是否已经有可用副本；
- 复制策略是只主线、尽力备份还是要求备份；
- 路由和 failover 何时可用；
- 诊断如何告诉 operator 下一步。

## 边界规则

- failover 必须显式开启；
- smart routing 至少需要两个 ready 来源；
- 静态资源不能回退到 Go origin；
- R2 主要是 object/CDN 线路，不是新 Web hosting 主路径；
- provider 写入、路由和回滚需要证据，不靠猜测。

## Milestone 1：资源能力模型

记录每个 storage/resource library 的容量、文件大小、批量限制、日上传限制、健康状态和能力说明。

## Milestone 2：副本状态与修复

引入对象副本状态，提供 `replicas`、`refresh-replicas`、`repair-replicas` 等命令，帮助 operator 看到对象是否真的在目标线路可用。

## Milestone 3：复制策略

用 `replication_policy` 明确上传后的副本要求：

- `primary_only`；
- `best_effort_backups`；
- `require_backups`。

## Milestone 4：智能路由形式化

定义 routing policy、region group、权重/优先级、健康过滤和候选选择逻辑。上线前必须能用 `route-explain` 说明一次请求为何选择某个来源。

## Milestone 5：显式 Failover 硬化

`-resource-failover` 必须 opt-in，且必须有 ready candidates。Worker failover 不应使用 Go origin 作为静态资源兜底。

## Milestone 6：Probe、浏览器检查和测速

`probe-site`、`site-doctor`、`cdn-doctor`、browser render probe 和真实场景回归逐步变成发布证据。

## Milestone 7：生产 Canary 矩阵

真实环境应覆盖：

- Cloudflare Static；
- `hybrid_edge`；
- AList/OpenList；
- R2；
- IPFS/Pinata；
- rollback/recovery/writeback；
- edge manifest 路由。

## 建议执行顺序

1. 先建立能力模型和副本状态。
2. 再做复制策略和修复命令。
3. 然后补智能路由解释和 failover 守卫。
4. 最后用 probe、browser render 和 canary 固化成熟度。

## v0.3 非目标

- 不做自动“用户无感切线”；
- 不做没有证据的 metadata-only provider 操作；
- 不把旧 R2 Web 路径扩成主流方向；
- 不跳过 dry-run 和 operator 确认。

## Definition Of Done

- 资源能力和副本状态可见；
- route/failover 边界有文档和测试；
- operator 有明确诊断命令；
- live canary 能证明关键路径；
- 后续重构可从清楚边界开始。
