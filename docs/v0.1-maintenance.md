# Super CDN v0.1.x Maintenance

Status: feature-frozen maintenance line.

`v0.1.x` is the stable internal line after `v0.1.0`. Do not add product features on this line. Use it for bug fixes, documentation, operational hardening and regression verification only. New product ideas belong in the next minor version.

## Scope Freeze

Allowed in `v0.1.x`:

- Bug fixes for existing site, bucket, auth, probe, storage and deployment behavior.
- Documentation updates and operational runbooks.
- Test coverage for behavior that already exists.
- Release packaging fixes.
- Safe config examples that describe existing fields.

Not allowed in `v0.1.x`:

- New storage providers.
- New public API surfaces unless required to repair existing behavior.
- New workspace management features beyond the existing default workspace.
- AList/OpenList automatic multi-origin failover.
- Cloudflare Static rollback/version cleanup workflows beyond the current conservative guards.
- Public SaaS/team administration UI.

## Quick Start On Ubuntu

Download the release:

```bash
mkdir -p ~/supercdn
cd ~/supercdn

curl -L -o supercdn-v0.1.0-linux-amd64.tar.gz \
  https://github.com/Purewo/super_cdn/releases/download/v0.1.0/supercdn-v0.1.0-linux-amd64.tar.gz

tar -xzf supercdn-v0.1.0-linux-amd64.tar.gz
sudo install -m 755 supercdn-v0.1.0-linux-amd64/supercdnctl /usr/local/bin/supercdnctl
```

For a single-user trusted machine, put root control-plane access in `~/.bashrc`:

```bash
cat >> ~/.bashrc <<'EOF'

# Super CDN root access
export SUPERCDN_URL="https://qwk.ccwu.cc"
export SUPERCDN_TOKEN="<server.admin_token>"
EOF

source ~/.bashrc
supercdnctl whoami
```

Expected root identity:

```json
{
  "root": true,
  "workspace_id": "default",
  "role": "owner"
}
```

## Common Workflows

Create and deploy an overseas Cloudflare Static site:

```bash
supercdnctl create-site -site demo -profile overseas -mode spa
supercdnctl deploy-site -site demo -dir ./dist -profile overseas -static-spa
supercdnctl probe-site -site demo -spa-path /movie/123
```

Create an overseas CDN bucket and upload an asset:

```bash
supercdnctl create-cdn-bucket -slug overseas-assets -types image,archive
supercdnctl upload-bucket -bucket overseas-assets -file ./poster.jpg -path images/poster.jpg -asset-type image -warmup
```

Create a domestic mobile CDN bucket and upload an asset:

```bash
supercdnctl create-mobile-cdn-bucket -slug mobile-assets -types image,document
supercdnctl upload-bucket -bucket mobile-assets -file ./poster.jpg -path images/poster.jpg -asset-type image -warmup
```

Invite a teammate without exposing root secrets:

```bash
supercdnctl invite-user -name alice -role maintainer
```

The teammate accepts the one-time invite token:

```bash
supercdnctl -server https://qwk.ccwu.cc -profile alice login -invite-token sci_xxx
supercdnctl -profile alice whoami
```

## Regression Checklist

Run local checks before every `v0.1.x` release:

```powershell
go test ./...
go vet ./...
```

Run production smoke checks after deployment:

```powershell
curl.exe -fsS https://qwk.ccwu.cc/healthz
.\bin\supercdnctl.exe -server https://qwk.ccwu.cc cloudflare-status -all
.\bin\supercdnctl.exe probe-site -url https://cyberstream-mobile-go.sites.qwk.ccwu.cc/ -spa-path /movie/123 -max-assets 10
```

Auth smoke:

1. `whoami` with root returns `root=true` and `role=owner`.
2. Root creates a short-lived `viewer` invite.
3. Invite acceptance returns a user API token.
4. Viewer `whoami` returns the smoke user and `role=viewer`.
5. Viewer write request such as `POST /sites` returns `403`.
6. Root revokes the smoke token.

Line smoke:

- Overseas Web: deploy or probe one Cloudflare Static site.
- Overseas CDN: create or reuse one R2-backed bucket and verify returned public/R2 URLs.
- Domestic CDN: create or reuse one mobile bucket and verify public URL plus Range GET behavior.
- Domestic Web: probe one mobile origin-assisted site and one SPA fallback path.

Domestic all-line smoke:

```powershell
.\bin\supercdnctl.exe resource-status -library repo_china_all
.\bin\supercdnctl.exe health-check -libraries repo_china_all -write-probe -force
.\bin\supercdnctl.exe create-domestic-cdn-bucket -slug local-all-cdn-smoke-<stamp> -line all -types document -cache-control "public, max-age=86400"
.\bin\supercdnctl.exe upload-bucket -bucket local-all-cdn-smoke-<stamp> -file .\README.md -path docs/readme-<stamp>.md -asset-type document -warmup
.\bin\supercdnctl.exe list-bucket -bucket local-all-cdn-smoke-<stamp> -prefix docs/
```

Expected result: `repo_china_all` health is `ok`, the bucket has `route_profile=china_all`, upload returns a public URL plus storage/CDN URL, warmup succeeds, the public URL can be read, and `list-bucket` returns the uploaded object.

## Production Rollback

Current production service:

```text
host: 166.0.198.218
unit: supercdn
install dir: /opt/supercdn
public URL: https://qwk.ccwu.cc/
origin bind: 127.0.0.1:8080
```

Before replacing binaries, create a dated backup under `/opt/supercdn/backups/` containing:

- `/opt/supercdn/bin/supercdn`
- `/opt/supercdn/bin/supercdnctl`
- `/opt/supercdn/config.json`
- `/opt/supercdn/data/supercdn.db*`

Manual rollback on the server:

```bash
BACKUP=/opt/supercdn/backups/<backup-dir>
systemctl stop supercdn
install -m 755 "$BACKUP/supercdn" /opt/supercdn/bin/supercdn
install -m 755 "$BACKUP/supercdnctl" /opt/supercdn/bin/supercdnctl
systemctl start supercdn
curl -fsS http://127.0.0.1:8080/healthz
systemctl status supercdn --no-pager
```

Only restore the SQLite database when the new binary performed an incompatible migration or wrote bad data. For ordinary binary rollback, leave the current DB in place unless the incident analysis says otherwise.

## Operational Notes

- Root access is `server.admin_token`; there is no separate username/password login.
- Keep root token off shared machines. Use `invite-user` for team members.
- Signed AList/OpenList and cloud storage URLs must stay redacted in shared logs.
- `cloudflare-status -all` is the fastest Cloudflare token/zone/R2 health check.
- AList/OpenList token refresh depends on configured username/password where available.
- If a hybrid edge site starts routing to expired signed URLs, use `refresh-edge-manifest` before rebuilding the whole site.
- If Cloudflare Static custom-domain verification fails, check DNS proxy mode and use one-level hosts under `qwk.ccwu.cc` for Worker Static Assets.
