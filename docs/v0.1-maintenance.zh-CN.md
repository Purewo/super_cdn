# Super CDN v0.1.x 维护手册

[English](v0.1-maintenance.md) | 简体中文

v0.1.x 是 feature-frozen 的内部稳定线。本手册只用于维护、回归和紧急修复，不作为新功能开发入口。

## 范围冻结

允许：

- 安全修复；
- 文档修正；
- 运维脚本修正；
- 回归测试补充；
- 明确不改变产品语义的小 bugfix。

不允许：

- 新 Web hosting 模型；
- 新 provider 集成；
- 智能路由、failover 或回滚语义变化；
- 大范围重构。

## Ubuntu 快速启动

典型步骤：

```powershell
go build ./cmd/supercdn
go build ./cmd/supercdnctl
SUPERCDN_ADMIN_TOKEN=change-me ./supercdn
```

另一个 shell：

```powershell
SUPERCDN_TOKEN=change-me ./supercdnctl doctor
SUPERCDN_TOKEN=change-me ./supercdnctl create-project -id assets
SUPERCDN_TOKEN=change-me ./supercdnctl upload -project assets -file ./README.md -path docs/readme.txt -profile overseas
```

## 常见工作流

- 创建项目：`create-project`。
- 上传静态对象：`upload`。
- 检查环境：`doctor`。
- 发布或查看站点时，优先参考当前版本 README 和命令文档，不要把 v0.1 当成新功能说明。

## 回归清单

维护 v0.1.x 时确认：

- Go 测试通过；
- 二进制可构建；
- `/healthz` 正常；
- `doctor` 不泄露 secret；
- 基础对象上传/读取可用；
- 文档没有把 v0.1 描述成当前主线。

## 生产回滚

v0.1.x 的回滚应保守处理：

- 先备份现有二进制和数据；
- 回滚到已验证 commit/tag；
- 验证 `/healthz`、`doctor` 和最小上传读取路径；
- 不在 v0.1.x 引入 Cloudflare provider-aware rollback 语义。

## 运维备注

v0.1.x 主要价值是保留最早可用基线。当前开发应从 `v0.5.0` 和 [refactor-plan.zh-CN.md](refactor-plan.zh-CN.md) 继续。

## 延后到后续版本

- 资源桶；
- 批量上传；
- Web hosting 正式边界；
- OpenAPI；
- audit events；
- CI release gate；
- Cloudflare Static / hybrid edge rollback。
